# 붉은사막 파밍 가이드 MVP

붉은사막 아이템 파밍 포인트를 맵 위에 표시하고, 위치 설명과 추천 동선, 획득조건, 선행조건, 능력/효과, 출처 영상까지 함께 보여주는 정적 웹앱 프로토타입입니다.

현재는 사용자가 제공한 유튜브 링크들을 바탕으로 실제 아이템/스킬/탈것 항목을 1차 반영해둔 상태입니다.

## 실행 방법

1. `index.html`을 브라우저에서 바로 엽니다.
2. 검색창, 카테고리 필터, 지역 선택으로 포인트를 좁힙니다.
3. 맵 마커나 하단 카드 목록을 누르면 오른쪽 설명 패널이 바뀝니다.
4. `좌표 보정` 버튼을 켜면 마커를 드래그하거나 슬라이더로 위치를 미세 조정할 수 있습니다.
5. 출처 카드의 `구간 열기`를 누르면 해당 유튜브 챕터 시작 시점으로 바로 이동합니다.
6. `보정 JSON 불러오기`로 저장해둔 좌표 패치를 다시 앱에 적용할 수 있습니다.

## GitHub Pages 배포

이 프로젝트는 `.github/workflows/deploy-pages.yml`이 포함되어 있어서, GitHub 저장소의 `main` 브랜치에 올리면 GitHub Pages로 자동 배포되게 준비돼 있습니다.

예상 주소 형식:

- `https://<github-username>.github.io/<repo-name>/`

배포 순서:

1. GitHub에서 새 저장소를 만듭니다.
2. 현재 폴더 전체를 저장소 루트에 업로드하거나 push합니다.
3. 기본 브랜치를 `main`으로 둡니다.
4. 첫 push 후 GitHub Actions의 `Deploy GitHub Pages` 워크플로가 실행됩니다.
5. GitHub 저장소의 `Settings > Pages`에서 소스가 `GitHub Actions`인지 확인합니다.

이 환경에는 현재 `git`와 `gh`가 없어서 자동 push까지는 못 했습니다. 저장소 URL이나 저장소 이름이 정해지면 그 기준으로 마지막 연결 절차를 이어갈 수 있습니다.

## 현재 상태

- 맵 이미지는 `assets/pywel-placeholder-map.svg`에 있는 샘플 아트입니다.
- 아이템/위치/조건/출처 데이터는 `data/guideData.js`에 있으며, 일부는 실제 유튜브 영상 기준으로 교체됐습니다.
- 현재 좌표는 `영상 챕터 + 설명문` 기준의 임시 배치가 포함되어 있습니다.
- 실제 붉은사막 파밍 가이드로 쓰려면 맵 캡처와 영상 프레임을 대조해 좌표를 확정하면 됩니다.
- 좌표 패치 예시는 `data/verified-coordinate-overrides.template.json`에 있습니다.
- 실제 반영용 기본 파일은 `data/verified-coordinate-overrides.json`입니다.

## 현재 반영된 영상 기반 항목

- 무료 스킬 해금 루트
- 마녀 성소 / 해적왕 모자 루트
- 입문 장비 + 금괴 스타트 루트
- 검은 운명 입수 루트
- 검은 운명 대체 상위 무기 후보
- 차가운 어둠의 판금 갑옷 세트
- 자석펫 영입 루트
- 특대형 가방 루트
- 공속 4 반지 세팅 루트
- 전설의 백마 로일러
- 전설의 흑마 로케이드
- 전설의 적토마 카모라

## 실제 게임 데이터로 바꾸는 법

### 1. 맵 교체

- 실제 맵 스크린샷이나 직접 제작한 지도 이미지를 준비합니다.
- 파일명을 유지하려면 `assets/pywel-placeholder-map.svg`를 같은 이름으로 교체하면 됩니다.
- 다른 파일명을 쓰고 싶으면 `index.html`의 `<img class="map-image">` 경로만 바꾸면 됩니다.

### 2. 포인트 추가

`data/guideData.js`의 `spots` 배열에 객체를 추가하면 됩니다.

```js

  coordinates: { x: 58, y: 63 }, // 맵 기준 퍼센트
  difficulty: "중간",
```

## 유튜브 기반 입력 워크플로

1. 영상에서 지역 랜드마크를 먼저 잡습니다.
2. 실제 맵 이미지 위에서 대략 좌표를 찍어 `coordinates`에 퍼센트로 기록합니다.
3. 영상에서 드롭 장면, 툴팁, 퀘스트 조건이 나온 시간대를 `cue`로 남깁니다.
4. `items` 안에 아이템 이름, 효과, 능력치, 획득조건, 선행조건을 분리해서 넣습니다.
5. 하나의 영상만으로 확정하기 애매한 정보는 `verification.status`를 `영상 교차확인 권장`으로 두고, `sources`를 2개 이상 붙입니다.
6. 패치나 버전 차이가 의심되면 `lastChecked` 날짜를 갱신합니다.

## 검증 단계 가이드

- `verification.status`는 아이템 정보 검증 수준입니다.
- `mapping.status`는 실제 맵 좌표 검증 수준입니다.
- `복수 영상 교차확인`은 같은 정보가 두 영상 이상에서 반복 확인된 경우입니다.
- `좌표 임시 배치`는 실제 게임 맵 프레임 대조 전의 연구용 마커입니다.
- `다중 위치 묶음`은 말 3마리나 무료 스킬처럼 여러 지점을 대표 마커 하나에 묶은 상태입니다.

## 좌표 보정 모드 사용법

1. 앱 상단의 `좌표 보정` 버튼을 켭니다.
2. 맵 위 마커를 직접 드래그하거나 상세 패널의 X/Y 슬라이더를 움직입니다.
3. 보정 좌표는 브라우저 `localStorage`에 저장됩니다.
4. 상세 패널의 `현재 좌표 JSON 복사`로 수정된 좌표 스냅샷을 추출할 수 있습니다.
5. `이 항목 초기화`는 선택한 마커만 되돌리고, `전체 보정 초기화`는 모든 로컬 수정값을 지웁니다.
6. 상단의 `보정 JSON 복사`는 현재까지 수정한 모든 마커 좌표를 한 번에 내보냅니다.
7. `보정 JSON 불러오기`는 같은 구조의 JSON 파일을 다시 읽어와 좌표와 로컬 검수 상태를 복원합니다.
8. 좌표를 최종 확정했다면 상세 패널의 `현재 좌표 확정 처리`를 눌러 로컬 확정 상태를 남길 수 있습니다.

## 데이터 반영 스크립트

브라우저에서 보정한 좌표를 실제 데이터 파일에 반영하려면 아래 순서로 진행합니다.

1. 앱에서 `보정 JSON 복사`를 눌러 내용을 저장합니다.
2. 저장한 내용을 `data/verified-coordinate-overrides.json`에 붙여 넣습니다.
3. 아래 명령으로 `guideData.js`에 반영합니다.

```powershell
node scripts/apply-coordinate-overrides.js
```

미리보기만 하려면:

```powershell
node scripts/apply-coordinate-overrides.js --dry-run
```

다른 입력 파일을 쓰려면:

```powershell
node scripts/apply-coordinate-overrides.js --input data/verified-coordinate-overrides.template.json
```

## 추천 입력 원칙

- 위치 정보와 아이템 정보는 분리해서 씁니다.
- 능력치는 영상 속 툴팁이나 인벤토리 화면이 보일 때만 확정합니다.
- 획득조건과 선행조건은 꼭 나눠 적습니다.
- 추정 정보는 `note`에 남기고 본문에는 확정값처럼 쓰지 않는 편이 안전합니다.
- 같은 아이템이 여러 위치에서 나오면 스팟을 나누고, 추후 아이템 중심 보기 화면을 추가하는 방식이 좋습니다.

## 추천 확장 순서

1. 실제 지역 맵 1장과 대표 파밍 포인트 10개만 먼저 입력합니다.
2. 아이템 이름, 효과, 조건, 출처 영상 시간을 `items`와 `sources`에 꼼꼼히 채웁니다.
3. 이후에는 드롭 확률, 추천 레벨, 패치 버전, 아이템 등급 필터를 붙입니다.
4. 데이터가 많아지면 `guideData.js`를 JSON 또는 간단한 CMS 구조로 분리합니다.
5. 그다음 단계에서 유튜브 링크 자동 수집기나 관리자 입력 화면을 붙입니다.
